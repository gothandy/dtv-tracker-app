# V2.1 Self-Service Booking, Cancellation and Spaces

## Purpose

This milestone completes the core self-service volunteer journey for the DTV Tracker App.

It is about moving from a promising frontend to a genuinely useful operational tool by allowing volunteers to understand availability, book themselves onto suitable sessions, and cancel when needed.

## Charity value

This milestone should deliver clear charity value by:

- reducing admin handling around volunteer sign-up and changes
- giving volunteers a clearer and more self-service path into sessions
- reducing reliance on Eventbrite for the normal booking journey without forcing its removal
- making session availability more understandable and therefore easier to communicate
- creating a stronger operational base for future volunteer-facing improvements

## Charity burden effect

This milestone is intended to reduce burden more than it adds.

It should:

- reduce manual back-and-forth around bookings and cancellations
- reduce dependence on one-off admin intervention for routine session changes
- keep Eventbrite available as a fallback rather than removing it too early
- keep paper registration as an on-the-day fallback where connectivity or devices are unavailable

It should avoid:

- creating a more complicated limits model than the charity can comfortably maintain
- making the Tracker the only viable way to manage volunteer attendance
- expanding into broader admin edit complexity unless it is needed to support the core volunteer journey

## What completion looks like

When this milestone is complete:

- a self-service volunteer can see whether a session is available to them
- a self-service volunteer can book onto a suitable session through the Tracker
- a self-service volunteer can cancel their booking through the Tracker
- spaces and availability are clear enough to be trusted in normal use
- the charity has reduced dependency on Eventbrite for routine bookings, while still keeping that option open
- paper registration remains a valid fallback for on-the-day operations

## What this milestone does not try to do

This milestone does not aim to:

- fully remove Eventbrite
- make the Tracker the sole operational path for volunteer attendance
- finalise every aspect of limits editing or advanced availability policy
- deliver broader categories, projects, push messaging, or content reuse features
- tackle the full maintainability, GDPR, and security backlog

## Key dependencies or constraints

- The volunteer-facing flow should be prioritised ahead of deeper admin editing complexity
- Limits logic should be treated carefully where raw stored values and derived values may diverge
- Existing fallbacks should be preserved, especially Eventbrite and paper registration
- The milestone should support the charity's wider resilience strategy, not narrow it
